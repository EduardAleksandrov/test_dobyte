## Алгоритм

Алгоритм полного перебора 13 чисел не эффективен по времени, поэтому используем алгоритм Разделения по суммам.

Основные идеи алгоритма

- Определение "красивого" числа:

Число считается красивым, если сумма его первых шести цифр равна сумме последних шести цифр.

- Разделение числа на две половины:

Мы рассматриваем 13-значное число как две половины по 6 цифр и одну неиспользуемую цифру (которая может быть любой из возможных 13 значений).

Первые 6 цифр и последние 6 цифр независимы в отношении суммы.

- Подсчет возможных комбинаций сумм:

Вместо проверки каждой возможной комбинации 13-значных чисел, мы заранее генерируем все возможные суммы для первых шести и для последних шести цифр.

Для каждой половины мы храним количество способов (комбинаций), при которых можно получить каждую сумму.
Использование словарей (или хеш-таблиц):

Мы используем ассоциативные массивы (или map в C++) для хранения:

Ключ: сумма (от 0 до 72, так как максимальная сумма шести цифр в тринадцатиричной системе равна 6×12=72).

Значение: количество способов (или комбинаций) для достижения этой суммы из 6 цифр.

- Сравнение сумм:

После того как подсчитаны все возможные суммы для двух половин, мы простым образом можем подсчитать количество красивых чисел, умножая количество комбинаций для одинаковых сумм из первой и второй половин.
